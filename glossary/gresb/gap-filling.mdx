Gap-filling refers to estimation methods used to fill missing meter readings so you can report complete annual data. GRESB accepts certain estimation approaches when actual data is unavailable.

<Note>
**How Abisko Expands from ESPM:** ESPM allows manual data entry but doesn't track estimation methodology. Abisko documents which values are estimated vs actual and applies GRESB-compliant estimation methods automatically.
</Note>

## Acceptable Gap-Filling Methods

| Method | When to Use | GRESB Compliant |
|--------|-------------|-----------------|
| **Proration** | Partial month readings | ✅ Yes |
| **Prior year same period** | Stable consumption patterns | ✅ Yes |
| **Weather normalization** | Climate-dependent loads | ✅ Yes |
| **Interpolation** | Single missing month | ✅ Yes |
| **Flat estimation** | No basis for calculation | ⚠️ Flag required |

## How Gap-Filling Works in Abisko

<Steps>
  <Step title="Identify gaps">
    **Data Quality** dashboard highlights properties with incomplete data
  </Step>
  
  <Step title="Review gap details">
    Click property → **Meters** → see specific date ranges missing
  </Step>
  
  <Step title="Select estimation method">
    Abisko suggests appropriate method based on gap size and data patterns
  </Step>
  
  <Step title="Apply and document">
    Estimated values flagged in exports so GRESB knows methodology used
  </Step>
</Steps>

<Tip>
**Best Practice:** Gap-fill early in the year when you still have time to chase down actual utility bills. Estimated data counts toward completeness but actual data is always preferred.
</Tip>
